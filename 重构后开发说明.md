# 数据分析看板开发说明

## 当前启动方式

统一从根目录启动：

```bash
streamlit run main.py
```

启动后：

1. 根页负责上传和管理业务数据源、打标文件。
2. `pages/` 下的页面会自动出现在 Streamlit 侧栏。
3. 各业务看板共享当前会话中的上传文件。

## 分层约定

- `main.py` 只负责共享文件上传与会话状态管理。
- `pages/` 只保留页面入口和页面编排。
- `app/core/` 放跨看板复用的基础能力。
- `app/dashboards/<dashboard_name>/` 放具体看板的配置、加工、服务和 UI 渲染。

## 新增一个业务看板的方法

1. 在 `app/dashboards/<dashboard_name>/` 下创建看板目录。
2. 新增 `config.py` 定义表名、字段名和展示列。
3. 按需要补充 `processor.py`、`service.py`、`tree_builder.py`、`ui.py`。
4. 在 `pages/` 下新增一个页面文件，例如 `08_投放概览.py`。
5. 页面文件只负责检查数据源、调用服务层和渲染 UI。

新增入口时不需要复制首页上传逻辑，也不需要修改已有页面。
