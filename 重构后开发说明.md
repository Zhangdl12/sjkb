# 素材分析看板开发说明

## 当前启动方式

统一从根目录启动：

```bash
streamlit run main.py
```

启动后：

1. 根页负责上传和管理共享 Excel 数据源。
2. `pages/` 下的页面会自动出现在 Streamlit 侧栏。
3. 各个业务看板共享当前会话中的同一份上传文件。

## 分层约定

- `main.py`
  - 只负责共享数据源上传与会话状态管理。
- `pages/`
  - 只保留页面入口，不写上传、缓存和主流程编排。
- `app/core/`
  - 放跨看板复用的框架能力。
  - `shared_source.py` 管理共享文件会话状态。
  - `loader.py` 负责 workbook 加载与 sheet 抽取。
  - `filters.py` 提供通用侧边栏筛选。
  - `page_runner.py` 负责看板页面统一执行流程。
- `app/dashboards/<dashboard_name>/`
  - 放具体看板的全部实现。
  - 包括页面配置、数据加载包装、业务加工、指标计算和 UI 渲染。

## 共享数据源会话约定

当前实现使用以下 `session_state` 键保存首页上传的数据源：

- `shared_source_name`
- `shared_source_bytes`
- `shared_source_token`

其中：

- `shared_source_name` 用于显示当前文件名
- `shared_source_bytes` 存放原始文件内容
- `shared_source_token` 用于缓存标识

## 新增一个业务看板的方法

1. 在 `app/dashboards/<dashboard_name>/` 下创建看板目录。
2. 至少新增一个 `config.py`，定义 `DashboardPageConfig`。
3. 如有需要，再补充该看板专属的 `loader.py`、`processor.py`、`metrics.py`、`ui.py`。
4. 在 `pages/` 下新增一个页面文件，例如 `03_投放概览.py`。
5. 页面文件中只导入看板配置并调用 `run_dashboard_page()`。

这样新增入口时，不需要复制首页上传逻辑，也不需要修改已有页面。
